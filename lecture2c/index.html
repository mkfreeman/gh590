UCD Overview

  • Design philosophy that prioritizes users at each stage of development
  • Aims to meet needs/wants of users (not change behavior)
  • Engages users for design, prototyping, testing (ideally)
  • Provides a framework for prioritization and evaluation

UCD Process

  • Identify target population for product (users)
  • Construct aggregate representations of common users (personas)
  • Describe contexts in which product is used (scenarios)
  • Detail specific tasks to accomplish (use cases)

Users

Identify users through:

  • Field research
  • Analysis of existing products
  • Outreach and collaboration
  • Prior knowledge and estimation

Users in global health

  • Health officials
  • Health providers
  • International NGOs
  • General public
  • Researchers
  • Educators
  • You

Personas

“Fictional characters that represent users with shared attitudes, goals, and behaviors”
  • Increased empathy
  • Tighter focus
  • Better communication
  • Avoid stereotypes
  • (Putnam 2009)

Persona Description

Typically include:

  • Name
  • Demographics (age, language)
  • Profession
  • Roles and responsibilities
  • Goals and tasks
  • Technological experience

Persona Sample

  • Name: Marie
  • Age: 32
  • Country: Japan
  • Profession: HIV Researcher
  • Role: Estimate intervention efficacy

Scenarios

“Story about how a user interacts with a product”
  • Provides tangible/memorable context for product
  • Helps define requirements for the product
  • Identifies potential challenges/issues

Global Health Scenarios

  • An HIV researcher is writing a paper and wants to get HIV statistics
  • An academic uses the visualization in a class
  • A health minister wants to learn about age-specific HIV trends in her country

Use Cases

“A use case defines the interactions between external actors and the system under consideration to accomplish a goal” -wiki
  • Detail system level requirements for steps in a scenario
  • Articulate process for an interaction
  • Can help limit out unrelated requests

Example Use Cases

  • Getting HIV numbers: mouseover element to retrieve value
  • Learning about different ages: need a menu to select age
  • Learning about a specific country: click the country

Summary

Design the right product for the right people

  • Identify and describe your users
  • Write scenarios that facilitate understanding product use
  • Detail use cases to articulate steps executed in a scenario
